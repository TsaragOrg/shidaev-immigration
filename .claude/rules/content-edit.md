# Правило — Правка контента сайта

Подгружается когда трогаю текст на страницах или в компонентах содержательной части.

---

## Compliance — стоп-сигналы

Полный список запретов — в `AGENTS.md` «Чего НЕ делать». Здесь — только специфика правки текста:

- **Цифры и факты сверять** с `docs/facts/project-facts.md` (телефон, email, адрес, бар-номер, contacts).
- **Educational claims** («trial-trained», «федеральные суды по всей стране») — точные формулировки, прошли согласование. Не «улучшать» по ходу.
- Поймала себя на «надо чуть переформулировать про admissions / pro hac vice / статистику» → **стоп**. Записываю в `docs/facts/open-questions.md`, спрашиваю Zouli.

---

## RU + EN parity — обязательное правило

Любая правка контента ВСЕГДА в обеих языковых версиях.

| Изменение | RU | EN |
|---|---|---|
| Слово/фраза в Hero | ✓ | ✓ обязательно |
| Новая секция | ✓ | ✓ обязательно |
| Удалить блок | ✓ | ✓ обязательно |
| CTA текст | ✓ | ✓ обязательно |
| meta description | ✓ | ✓ обязательно |
| Картинка alt | ✓ | ✓ обязательно |

**Если поменяла только RU — задача НЕ сдана.** Это баг parity (см. `docs/04-функции/i18n.md`).

Структурное равенство ≠ дословный перевод. EN — это самостоятельная версия по смыслу. Не калька.

## Тон контента

| Аудитория | Регистр |
|---|---|
| Конечный пользователь (читатель сайта) | **Простой, ясный**. Без жаргона. Без юр-канцелярита там где можно объяснить понятно. |
| Zouli (в общении со мной, не в контенте) | Инженерный регистр |

В сайте — **не клиентский маркетинг**, не «лидеры рынка», не «передовые технологии». Сухо, точно, по делу. Тон Jacob — профессиональная сдержанность.

---

## Перевод — по смыслу, не по слову

Эталон: хорошее меню в ресторане, хорошая редакторская статья.

Тест: прочитай вслух. Если звучит как русский переведённый Google'ом — переписать.

| Плохо (калька) | Лучше (по смыслу) |
|---|---|
| «We provide comprehensive immigration services» | «Защита иммигрантов в иммиграционных судах» |
| «Записаться на консультацию» → «Sign up for consultation» | «Book a consultation» |
| «Свежая статья» → «Fresh article» | «Latest» / «From the journal» (контекст решает) |

## Что меняется по запросу правок

- **Тексты** — да
- **Фото** — да (с заглушками от Zouli, временные)
- **Названия кнопок** — да (но в рамках трёх типов из дизайн-системы)
- **CTA назначение** — да

## Что НЕ меняется по запросу правок

Это про целостность дизайна. См. также `.claude/rules/design-system.md`.

- ❌ **Структура** страницы (порядок секций)
- ❌ **Сетка** (grid system, контейнеры)
- ❌ **Цвета** базовой палитры
- ❌ **Шрифты**
- ❌ **Тип кнопок** (только из трёх существующих)

Если запрос требует изменить структуру/сетку/палитру — это **отдельная задача с обсуждением**, не «правка по ходу».

## Перед коммитом контент-правки

Чеклист:

- [ ] Поменяла в RU
- [ ] Поменяла в EN
- [ ] Compliance-стопы не задеты (admissions, дела, гарантии, legal)
- [ ] Тон выдержан (простой для пользователя, точный для Jacob)
- [ ] Перевод по смыслу, не калька
- [ ] Структура страницы не сломана
- [ ] Не ввела новые цвета/шрифты/типы кнопок
- [ ] Mobile-вид проверен (хотя бы мысленно — переполнение, переносы)

Если что-то не сходится — НЕ коммитить.
