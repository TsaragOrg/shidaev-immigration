## ✅ Принятые решения по архитектуре (информационно, ответа не требуют)

Это решения, которые Zouli приняла сама на основе исследования рынка, обсуждения с инженером и анализа дизайн-системы. Перечислено, чтобы посредник и Jacob были в курсе. Любое из них можно пересмотреть, если Jacob будет против.

### A. CMS — Sanity

Headless CMS на бесплатном плане. Free tier с большим запасом для блога одного юриста (1 статья/мес, 50 статей в год). На handoff'е проект полностью передаётся в собственность Jacob'а (transfer ownership) — на стороне Zouli ничего не остаётся.

### B. Архитектура контента — document-level localization

Каждая статья — отдельный документ в Sanity со своим языком (RU или EN). Связанные переводы ссылаются друг на друга. Jacob выбирает язык **при создании каждой статьи** — система автоматически кладёт статью на правильную языковую версию сайта.

Гибкость:
- Если Jacob делает только RU — статья появляется на /blog
- Если делает только EN — на /en/blog
- Если делает оба и связывает — на обеих версиях с кнопкой переключения

Не требует выбора режима заранее. Архитектура работает в любом сценарии.

### C. Главная страница блога — list layout + табы категорий

- Список в одну колонку (editorial стиль, без сетки карточек)
- Три таба сверху: [Все] [Полезно знать] [Из практики]
- Фильтр по табу без перезагрузки страницы
- Переход на hybrid layout (featured + grid) — когда статей станет 20+

### D. Карточка статьи в списке — без фотографий

Структура: eyebrow (категория) · reading time · язык / EB Garamond заголовок / deck-строка / дата.

Editorial-стиль держится на отсутствии фото — добавление картинок-обложек сломает визуальный язык. Решение зафиксировано **на будущее тоже** — не добавляем фото даже когда «появятся».

### E. Страница статьи — паттерны длинного чтения

- EB Garamond для body, 17-18px мобиле, 19-20px desktop
- Line-height 1.7, max-width 660px (комфортно глазу)
- Складное оглавление в начале
- Progress bar 2px бронза сверху viewport
- Кастомные блоки в теле: callout «Важно знать», pull-quote, чек-лист, ссылка на прецедент, inline-картинка
- 2 связанные статьи в конце (из той же категории)
- Share: «Скопировать ссылку» + native share (без Twitter/FB кнопок)
- Кнопка «Read in English» / «Читать по-русски» если перевод есть

### F. Bilingual — поведение на сайте

На EN-версии сайта статьи, доступные только на русском, показываются с плашкой «Russian only» (а не скрываются). Клик ведёт на RU-версию. Это уважает двуязычность аудитории Jacob'а (иммигранты из RU-стран часто читают оба языка).

### G. Кастомные блоки контента (Portable Text)

В редакторе Jacob'а будут готовые блоки, которые он вставляет в текст одной кнопкой:

- **Callout** — плашка с бронзовым border-left для важных фактов/дедлайнов
- **Pull-quote** — крупно выделенная цитата (только раз на статью)
- **Чек-лист** — список «что взять / что подготовить»
- **Case reference** — карточка прецедента с цитатой
- **Inline-image** — картинка в тексте с подписью

Все блоки pre-styled, единообразны, не позволяют Jacob'у сломать дизайн (как в WordPress).

### H. Source files и accounts ownership

Все production-аккаунты создаются на Jacob'а / на его organization:
- Sanity organization → Jacob
- Vercel team → Jacob
- Calendly → Jacob
- Stripe → Jacob
- Domain → Jacob (уже у него)
- GitHub repo → переводится в его аккаунт на handoff

Zouli — приглашённый коллаборатор на время разработки. После handoff'а у Zouli не остаётся ни одного production-доступа (кроме как референса проекта в её GitHub для портфолио).
