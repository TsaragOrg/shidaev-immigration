# Доступность (a11y)

## Целевой уровень

**WCAG 2.1 уровня AA** (декларировано в Accessibility Statement на `/accessibility` и `/en/accessibility`).

US юрисдикция → **ADA Title III** требования покрываются WCAG AA.

## Что реализовано

| Элемент | Где |
|---|---|
| Контраст текста ≥ 4.5:1 | Через `00-tokens-base.css`, проверено перед сдачей |
| Контраст крупного текста ≥ 3:1 | Через `00-tokens-base.css` |
| Alt-тексты на изображениях | См. `feedback_alt_atributy` правки, hero фото имеют alt |
| Touch targets ≥ 44×44px | Кнопки, ссылки на CTA |
| Input font-size ≥ 16px | iOS не зумит |
| `lang` атрибут на `<html>` | Корректный для каждого роута |
| Логическая иерархия заголовков | h1 → h2 → h3 без скачков |
| Один `<h1>` на страницу | Проверено |
| `focus-visible` сохранён | Не убираем outline без замены |
| Hover only на hover-устройствах | `@media (hover: hover)` (см. `19-touch-hover-reset.css`) |
| Иконки-кнопки имеют label | `aria-label` где нужно |
| Decorative SVG | `aria-hidden="true"` |

## Что проверяется на этапе 6.5

- Lighthouse Accessibility ≥ 90
- axe DevTools — критических ошибок 0
- Кроссбраузерность включая iOS Safari

## Чего НЕ требует Шидаев

- RGAA (Франция) — клиент US, не применимо
- EAA (EU Accessibility Act 2025) — клиент US, не применимо
- Видео/аудио субтитры — на сайте нет видео-контента
