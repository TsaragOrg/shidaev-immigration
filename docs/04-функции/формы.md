# Формы

## Решение: site-owned backend форм на сайте НЕТ

Сознательное решение проекта. Основной сбор данных через **Calendly** (имя/email/тип консультации) + прямые контакт-каналы (`mailto:`, `tel:`, WhatsApp).

На `/contact` есть модалка «Написать письмо», но она не является backend'ом сайта: отправка идёт во внешний endpoint из `NEXT_PUBLIC_CONTACT_FORM_ENDPOINT`. Сейчас там placeholder Formspree (`https://formspree.io/f/YOUR_FORM_ID`), поэтому форма не production-ready до выбора сервиса.

## Почему

- Меньше backend-инфраструктуры (нет API routes для приёма)
- Меньше юр-обязательств по обработке персданных (RGPD/CCPA проще)
- Calendly уже собирает нужное (Имя, Email, Время) перед консультацией
- Нет собственного backend/API route → меньше риска хранить или обрабатывать данные неправильно

## Какие данные собирает Calendly

- Имя
- Email
- Телефон (опционально)
- Тип консультации
- Слот (дата + время)
- Тайм-зона (auto-detected)

Никакие данные с этого сайта в нашу систему не идут, пока не настроен внешний form endpoint. Сайт = триггер Calendly popup + прямые ссылки + optional external form endpoint.

## Если в будущем понадобится своя форма

- Создать spec `04-функции/<новая-форма>.md` ПЕРЕД кодом, если появляется site-owned backend/API route.
- Добавить выбранную библиотеку (React Hook Form / Formik / собственный state) в `02-стек.md`
- Добавить выбранный сервис отправки (Resend / SendGrid / Vercel form) в `02-стек.md`
- Юридическая обвязка: обновить `04-функции/юридические-страницы.md` (Privacy с упоминанием формы)
