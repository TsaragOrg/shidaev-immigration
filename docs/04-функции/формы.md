# Формы

## Решение: custom-форм на сайте НЕТ

Сознательное решение проекта. Сбор данных через **Calendly** (имя/email/тип консультации) + прямые контакт-каналы (`mailto:`, `tel:`, WhatsApp).

## Почему

- Меньше backend-инфраструктуры (нет API routes для приёма)
- Меньше юр-обязательств по обработке персданных (RGPD/CCPA проще)
- Calendly уже собирает нужное (Имя, Email, Время) перед консультацией
- Нет fallback-формы → нет риска что отправится в никуда

## Какие данные собирает Calendly

- Имя
- Email
- Телефон (опционально)
- Тип консультации
- Слот (дата + время)
- Тайм-зона (auto-detected)

Никакие данные с этого сайта в нашу систему не идут. Сайт = триггер Calendly popup.
