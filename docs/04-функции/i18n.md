# Feature Spec — Языковое переключение (RU ⇄ EN)

Билингв сайт с переключением языка через явный контрол в шапке.

---

## Назначение

Дать пользователю возможность переключиться между русской и английской версией любой страницы, не теряя контекст.

## Подход

- **Раздельные роуты, не runtime-локализация.** RU = корневой путь (`/`, `/about`, `/blog/[slug]`), EN = `/en/[...]`.
- **Каждая страница существует в обеих версиях явно** — отдельный page.tsx, отдельный текст, отдельные meta.
- Это **не next-intl**, а ручное зеркало.

## Кто пользуется

- Русскоязычный пользователь, по умолчанию заходит на RU.
- Англоязычный пользователь, заходит через EN-ссылку или переключает явно.
- Family/friend которому переслали ссылку на «не своём» языке.

## Что видит пользователь

### Контрол переключения

- Расположение: в шапке (header), фиксирован.
- Визуал: компактный (RU / EN), не флаги.
- Состояние: активный язык подсвечен.

### Поведение

1. На RU-странице кликает «EN» → переход на тот же логический путь, но с префиксом `/en/`.  
   Пример: `/about` → `/en/about`.

2. На EN-странице кликает «RU» → переход на корневой путь без префикса.  
   Пример: `/en/blog/article-slug` → `/blog/article-slug`.

3. Переход = жёсткий навигейт на новый URL (не SPA-смена строки).

4. Скролл сбрасывается на верх (новая страница = новое чтение).

## Языковое зеркалирование страниц

Все статические страницы зеркалятся: `/about` ↔ `/en/about`, `/blog/[slug]` ↔ `/en/blog/[slug]`, и т.д.

Полный список маршрутов — `docs/spec.md` § 4 «Состав сайта».

**Правило parity:** если страница есть в RU — должна быть в EN. И наоборот. Расхождение = баг.

## Блог-публикации — особый случай

Категории и публикации идут через Sanity. У каждого поста:

- **Английский — основной язык.** Title, body, excerpt — на EN.
- **Русский — добавочный.** При наличии — показывается на RU-стороне. При отсутствии — fallback на EN с пометкой/без.
- Slug **общий** для обеих версий (англоязычный), не локализуется.
- Для публикаций автор не синхронизирует slug вручную: первая языковая версия имеет slug, перевод выбирает `Translation of`, а сайт использует slug связанной статьи.

Решение зафиксировано в схеме Sanity (`src/sanity/`).

## Что НЕ должно происходить

- Переключатель НЕ ведёт «на главную RU» когда пользователь был на `/en/contact`. Должен вести на `/contact` (тот же путь, другой язык).
- Если перевод страницы отсутствует — НЕ показывать «404» без объяснения. Должен быть осмысленный fallback (для блога — английская версия с notice).
- Контент НЕ должен «терять» язык при навигации по сайту — кликнул в EN-меню → должен остаться в EN до явного переключения.

## Что считается «равенством» страниц

Parity = **структурное равенство**, не дословное:

- Те же секции
- Тот же порядок
- Те же CTA на тех же местах
- Те же intent точки

Но **текст** адаптируется под язык — не калька с русского на английский. Перевод по смыслу (см. `feedback_translation_meaning_not_literal` в правилах).

## Исключительные случаи

| Случай | Поведение |
|---|---|
| Пользователь зашёл прямо на `/en/contact` | Показываем EN-версию, переключатель показывает «RU» как доступный |
| Блог-пост без RU перевода | На RU-стороне — показываем EN-версию (graceful fallback), переключатель есть |
| Запросил `/en/несуществующая-страница` | 404 (кастомная страница `/en/not-found`) |
| OG/meta теги | На каждой языковой версии — meta на своём языке + `alternate` ссылка на другой |
| sitemap.xml | Включает обе версии всех страниц + `hreflang` пары |

## SEO

- `<link rel="alternate" hreflang="ru" />` и `hreflang="en"` на каждой странице.
- `lang` атрибут на `<html>` соответствует языку текущей страницы.
- meta description и og:* — на языке страницы.
- Sitemap — обе версии равноправно.

## Контент-менеджмент при изменении

Когда меняется текст на RU:
- **Обязательно** меняется EN. Не «потом», не «когда вспомним». Сразу.
- Это правило → `.claude/rules/content-edit.md`.

Когда меняется структура (порядок секций, добавляется блок):
- Меняется в обеих версиях.
- Если не успеваешь в одной — задача не сдана.

## Где живёт реализация

- Header переключатель: `src/components/layout/Header.tsx`
- RU роуты: `src/app/(site)/[...]/page.tsx`
- EN роуты: `src/app/(site)/en/[...]/page.tsx`
- Sanity схема постов с локализацией: `src/sanity/`
