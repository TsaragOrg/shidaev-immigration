# Выбор темы обращения (intent grid)

## Назначение

На `/contact` пользователь выбирает с каким вопросом он обращается. Это позволяет Jacob/Maga понять контекст перед звонком/email, а пользователю быстро найти подходящий канал.

## Кто пользуется

| Роль | Что делает |
|---|---|
| Новый клиент | Выбирает тему → видит CTA «связаться» с правильным контекстом |
| Family/friend клиента | Выбирает тему «помощь близкому» → видит CTA |
| Возвращающийся клиент | Выбирает «вопрос по моему делу» → видит CTA |

## Когда вызывается (триггеры)

| Триггер | Откуда |
|---|---|
| Открытие `/contact` | Хедер / footer / любой CTA «связаться» |
| Скролл к секции intent grid на `/contact` | После hero страницы /contact |

## Что видит пользователь

Сетка карточек с темами:
- Asylum / убежище
- Removal defense / защита от депортации
- Federal litigation / федеральные суды
- Detained / помощь задержанным
- Family member's case / помощь близкому
- Other / другой вопрос

Каждая карточка:
- Иконка/индикатор (без рисованного декора)
- Заголовок темы
- Короткое описание
- При клике/hover на десктопе — раскрытие или переход к CTA

После выбора темы:
- Открывается соответствующий блок ниже (Calendly / WhatsApp / email)
- Или пользователь скроллит к универсальной booking-секции

## Что НЕ должно происходить

- Не **блокировать** выбором — пользователь должен мочь обратиться без выбора темы
- Не показывать «обязательный» индикатор у выбора
- Не разделять CTA по темам слишком сильно — основной канал один (Calendly)

## Исключительные случаи

| Случай | Поведение |
|---|---|
| Пользователь не выбирает тему | Universal CTA «Записаться на консультацию» внизу страницы доступен всегда |
| Тема «срочно/задержали родственника» | Возможен emphasis на телефон + WhatsApp как primary, а не Calendly |
| Mobile | Карточки в одну колонку, scroll-friendly |
