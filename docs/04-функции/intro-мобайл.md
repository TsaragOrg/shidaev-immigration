# Мобильная intro-секция

## Назначение

Только на мобайле: дополнительная intro-секция под Hero показывает портрет Jacob и краткое описание. На десктопе портрет уже в Hero, секция скрыта (`display: none`).

Решение принято на этапе 4 (v18.3) — на мобайле в Hero не помещается портрет с CTA одновременно, поэтому портрет вынесен ниже отдельной секцией.

## Кто пользуется

Любой посетитель главной страницы с устройством с экраном `< 768px`.

## Когда вызывается (триггеры)

Автоматически при загрузке `/` (RU) или `/en` на мобильном устройстве. Десктоп не показывает.

## Что видит пользователь

- Фото Jacob (cropped под мобильную композицию)
- Имя крупно
- Короткий descriptor: «Иммиграционный адвокат в Калифорнии»
- Возможно короткий lead-параграф

## Что НЕ должно происходить

- НЕ показываться на десктопе (CSS `display: none` на breakpoint)
- НЕ дублировать CTA из Hero — это идентификация, не повторный призыв
- НЕ занимать всю секцию выше fold — Hero CTA важнее

## Исключительные случаи

| Случай | Поведение |
|---|---|
| iPad portrait | По дефолту mobile breakpoint срабатывает — показывает intro. Acceptable. |
| Landscape mobile | Может занимать больше места по горизонтали — проверить composition |
| iOS Safari top-bar | Не должен перекрывать (см. `reference_ios_safari_top_bar` правило) |

## Где живёт реализация

- **Компонент:** Включён в `src/components/pages/HomePage.tsx` или в Hero (проверить)
- **Стили:** `17-intro.css`
- **Видимость:** `@media (max-width: ...)` управляет `display`
