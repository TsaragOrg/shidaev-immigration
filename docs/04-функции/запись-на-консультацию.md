# Feature Spec — Calendly Booking

Запись на консультацию к Jacob через виджет Calendly.

> Это spec поведения. Не техника, не страницы, не кнопки. Только что функция делает.

---

## Назначение

Позволить посетителю выбрать слот и записаться на консультацию без email-переписки, без звонка, без формы заявки на сайте.

## Кто пользуется

- Потенциальный клиент готовый говорить с адвокатом.
- Family/friend записывающий консультацию для близкого.

## Когда вызывается

Calendly popup открывается напрямую из **Hero главной страницы** и из **CTA на `/contact`**. Остальные CTA сайта (/about, /practice, blog, final CTA) ведут на `/contact`, где пользователь видит цену, каналы связи и кнопку Calendly.

| Триггер | Откуда |
|---|---|
| Hero CTA «Записаться» / "Book a consultation" | Home Hero |
| Кнопка «Записаться» в booking-секции `/contact` | ContactPage |
| Кнопка fallback (которая показывает URL Calendly) `/contact` | ContactPage |
| | |
| CTA с FinalCTA/About/Practice/Blog | Не открывает popup. **Ссылка** ведёт на `/contact`. |

Поток быстрого пользователя: главная → кликает Hero CTA → popup Calendly открывается поверх главной → выбирает дату.

Поток через подробности: другие CTA → `/contact` → пользователь видит цену, прямые каналы связи и CTA Calendly → popup открывается.

## Что видит пользователь

1. Кликает CTA «Записаться» / «Book consultation».
2. Если клик был на `/contact`, перед CTA видит цену консультации: `30 минут — $50` (или значение из env).
3. Открывается **popup поверх текущей страницы** (не редирект на `/contact`, не переход на новую страницу).
4. В popup — встроенный календарь Calendly: типы консультаций, доступные слоты, форма ввода данных.
5. После брони — Calendly показывает confirmation внутри popup.
6. Пользователь закрывает popup → остаётся на той же странице где был.

## Что НЕ должно происходить

- Popup НЕ блокирует прокрутку страницы под собой больше необходимого.
- Popup НЕ открывается случайно (только по клику CTA).
- На мобайле popup НЕ ломает layout — закрывается крестиком, возвращает на исходную позицию.
- Если Calendly недоступен (offline, блокировка скриптов) — пользователь НЕ остаётся без альтернативы (см. fallback).

## Данные пользователя

Что собирает Calendly в форме:
- Имя
- Email
- Телефон (опционально)
- Тип консультации (выбор из настроенных в Calendly)
- Слот (дата + время)
- Тайм-зона (auto-detected)

Что НЕ собираем на стороне сайта:
- Никакие данные через сайт-формы не идут. Сайт = только триггер popup.

## Уведомления

- Владелец календаря получает уведомление о новой записи через Calendly (email + Google Calendar).
- Клиент получает confirmation email от Calendly с calendar invite.
- На сайте после записи — ничего не сохраняем.

> ⚠️ **Сейчас календарь = Zouli (`calendly.com/zulihan1993/30min`)** — это тестовая настройка. Записи приходят Zouli, не Jacob. Перед переключением сайта на боевые контакты — заменить `NEXT_PUBLIC_CALENDLY_URL` на календарь Jacob в Vercel env и локальном `.env.local`.

## Fallback (если Calendly не работает)

Если Calendly widget не успел загрузиться или заблокирован, CTA открывает `NEXT_PUBLIC_CALENDLY_URL` в новой вкладке.

На `/contact` рядом с CTA остаются альтернативные каналы:

- Phone: `+1 (424) 558-4141` (клик-в-звонок)
- Email: `info@shidaev.com` (mailto)
- WhatsApp: тот же номер (срочный канал)

Это **всегда видно**, не прячется. Calendly = удобный путь, не единственный.

## Исключительные случаи

| Случай | Поведение |
|---|---|
| Slot не доступен (все занято на ближайшие недели) | Calendly сам показывает следующие доступные слоты. Сайт не вмешивается. |
| Пользователь закрыл popup без записи | Возвращается на страницу, состояние сохранено. Никаких «вы уверены?». |
| Пользователь на медленной сети | Loading state от Calendly. Не блокируем UI. |
| iOS Safari блокирует third-party iframe | Fallback — открыть Calendly как новую вкладку (deferred — пока не реализовано). |
| Adblocker блокирует calendly.com | Пользователь видит fallback-каналы (phone/email/WhatsApp). |

## Языковая локализация

- На RU-странице popup открывается с RU-локалью Calendly (если поддерживается).
- На EN-странице — EN-локаль.
- Если RU не поддерживается виджетом — EN дефолт (мягкий fallback).

## Что нужно от Calendly

- Аккаунт с настроенными типами консультаций.
- Embed-snippet для popup-режима.
- URL события у Jacob.

Конфиг живёт в env:

- `NEXT_PUBLIC_CALENDLY_URL` — URL события Calendly без widget-параметров, например `https://calendly.com/<jacob-handle>/30min`.
- Если переменная не задана, сайт использует временный demo fallback `https://calendly.com/zulihan1993/30min`, чтобы preview не ломался до handoff.
- Widget-параметры `primary_color=A8894A` и `hide_gdpr_banner=1` добавляет код автоматически.
- `NEXT_PUBLIC_CONSULTATION_PRICE_USD` — публичная цена 30-минутной консультации в USD, сейчас `50`.
- Если переменная не задана или невалидна, сайт использует fallback `$50`.
- Эта переменная меняет только отображение на сайте. Цена в Calendly/Stripe/LawPay настраивается отдельно в соответствующем сервисе.

## Метрики (если когда-то понадобятся)

- Сколько раз открывали popup
- Сколько из них записалось

Пока **не отслеживаем** — analytics не подключен (см. `docs/state.md`).

## Где живёт реализация

- Конфиг: `.env.local` / Vercel env `NEXT_PUBLIC_CALENDLY_URL` и `NEXT_PUBLIC_CONSULTATION_PRICE_USD`; пример — `.env.example`.
- Нормализация URL и widget-параметры: `src/lib/calendly.ts`.
- Нормализация и формат цены: `src/lib/consultation.ts`.
- Reusable assets: `src/components/booking/CalendlyWidgetAssets.tsx` — загружает Calendly CSS/script.
- Reusable CTA: `src/components/booking/CalendlyBookingButton.tsx` — открывает popup или fallback URL.
- Компонент: `src/components/home/Hero.tsx` — Hero CTA открывает Calendly popup напрямую.
- Компонент: `src/components/pages/ContactPage.tsx` — contact CTA открывает тот же Calendly popup.
- Стили booking-секции: `src/styles/25-contact-booking.css`
- Calendly widget script + CSS подгружаются через `<link>` и `<Script>` в JSX (`assets.calendly.com/...`)
- Где открывается popup: Hero главной (`/`, `/en`) и contact (`/contact`, `/en/contact`).
- С каких страниц CTA **ведут на `/contact`**: FinalCTA, /about, /practice, /blog — через обычный `<Link>`.

## Legacy замечание

В `src/styles/29-book-main.css` есть селектор `.calendly-inline-widget` — это **мёртвый код** от удалённой страницы `/book.html`, где Calendly был inline-виджетом. Текущая реализация — popup, эти стили не применяются. Подлежит чистке вместе с остальным book-* legacy CSS (см. `docs/state.md`).
