# Блог

## Назначение

Образовательный контент для аудитории. Jacob/Maga публикуют статьи о иммиграционном праве через Sanity Studio. Посетители читают на сайте.

## Кто пользуется

| Роль | Что делает |
|---|---|
| Jacob / Maga | Пишут и публикуют статьи через Sanity Studio (`/studio`) |
| Посетитель | Читает на `/blog` (список) и `/blog/[slug]` (статья) |
| Поисковики | Индексируют посты для SEO |

## Когда вызывается (триггеры)

| Триггер | Откуда |
|---|---|
| Клик «Свежая статья» на главной | Секция Journal главной → `/blog/[slug]` |
| Клик «Блог» в footer | Любая страница → `/blog` |
| Клик карточки на `/blog` | `/blog` → `/blog/[slug]` |
| Прямой ввод URL | браузер |

## Что видит пользователь

### `/blog` (индекс)
- **Featured post** — одна свежая публикация крупным баннером
- **Medium grid** — следующие 4-6 публикаций средними карточками
- **Text list** — остальные публикации компактным списком (BBC-формат)
- Empty state: если нет публикаций — осмысленная заглушка

### `/blog/[slug]` (статья)
- Заголовок, дата, категория
- Body content (Portable Text из Sanity)
- JSON-LD для SEO (`ArticleJsonLd.tsx`)
- Языковая alternate-ссылка (если перевод есть)

## Контент-модель (Sanity)

Тип `post`:
- title
- slug (общий для RU+EN; Studio допускает одинаковый slug в разных языках)
- language (`ru` / `en`; каждая версия — отдельный документ)
- translation (reference на статью в другом языке)
- category (одна из 7)
- date
- excerpt
- body (Portable Text)
- featured image (опционально)
- SEO title + description

Тип `category`: 7 категорий (см. `scripts/seed-categories.ts`).
Тип `author`: один автор Jacob Shidaev (см. `scripts/seed-author.ts`).

## Что НЕ должно происходить

- Не создавать дубли категорий — список зафиксирован в seed-скрипте
- Не заставлять автора вручную синхронизировать slug между переводами — при Publish slug копируется из связанной статьи
- Не блокировать удаление статьи из-за связанного перевода — delete-action сначала снимает `translation` у другой версии
- Не показывать незапубликованные посты (draft) — фильтр по `_id` без `drafts.`
- Не показывать пустой блог как «ошибку» — должна быть empty state
- Не терять SEO при отсутствии перевода — fallback на EN с notice

## Исключительные случаи

| Случай | Поведение |
|---|---|
| Sanity недоступен | Кешированная страница (ISR) показывает последние известные посты |
| Slug не найден | 404 с возможностью вернуться на `/blog` |
| Перевод отсутствует | Показываем EN-версию + (опционально) notice «переведено на EN» |
| Удаляют одну версию перевода | Studio снимает `translation` reference с другой версии, затем удаляет выбранную статью |
| Картинка не загрузилась | Карточка показывается без картинки (см. PostCard поддерживает оба варианта) |
| Пустой блог | Empty state с CTA «вернуться на главную» |

## i18n

- EN — основной язык поста (обязательно)
- RU — добавочный (опционально)
- Slug общий, не локализуется
- При связывании через `Translation of` publish-action автоматически синхронизирует общий slug и обратную reference-ссылку
- Категория EN основной + RU добавочный

См. `docs/04-функции/i18n.md`.
