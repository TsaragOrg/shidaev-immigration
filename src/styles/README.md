# src/styles/ — CSS-split

Основа стилей сайта. 38 файлов, по одному на секцию или точечный override.

Параллельно в проекте подключен Tailwind v4 (через `@import "tailwindcss"` в `src/app/globals.css`). Это **гибрид по правилу** — описан ниже.

---

## Когда что использовать

| Что нужно сделать | Куда писать |
|---|---|
| Новая секция / страница / переиспользуемый блок | **Сюда.** Новый файл `NN-name.css` + импорт в `globals.css` |
| Сложный grid с именованными областями | **Сюда** (читаемее чем Tailwind arbitrary) |
| `@keyframes` анимация | **Сюда** (`15-animations.css` или новый файл секции) |
| Mobile-only поведение секции | **Сюда** (`18-hero-mobile.css` — пример) |
| Global responsive overrides | **Сюда** (`16-responsive-global.css`) |
| Тонкий iOS Safari хак | **Сюда** (литеральный HEX в каркасе, см. `02-header.css`) |
| | |
| Мелкая утилита внутри JSX — gap, padding, text-size в одной точке | **Tailwind** (`className="gap-4 p-6 text-sm"`) |
| Spacing между двумя элементами без переиспользования | **Tailwind** |
| Quick layout правка в одиночном компоненте | **Tailwind** |

---

## Правила

- **Один файл = одна область ответственности.** Не сваливать несколько секций в один файл.
- **Имя:** `NN-name.css` где `NN` — номер по порядку каскада.
- **Порядок импортов в `globals.css` ОБЯЗАТЕЛЬНО** соответствует номерам. Перестановка ломает специфичность (особенно `16-responsive-global` перебивает базовые).
- **Tailwind импортится первой строкой** → его утилиты имеют меньшую специфичность чем правила в этих файлах. CSS-split побеждает по умолчанию.
- **Не дублировать.** Если стиль уже есть в файле — не делать Tailwind-альтернативу в JSX. И наоборот.
- **Новые цвета / spacing** → сначала переменная в `00-tokens-base.css`, потом использование. Не хардкодить.
- **Mobile-first.** Базовый стиль мобайл, десктоп через `@media (min-width: …)`.
- **Hover только в `@media (hover: hover)`** — иначе залипает на touch.

---

## Что НЕ делать

- ❌ Стилизованный `<a>` вместо одной из трёх кнопок (primary/secondary/tertiary)
- ❌ Inline `style={{...}}` в JSX (кроме редких динамических значений)
- ❌ CSS-in-JS (`styled-components`, `emotion`) — в стеке нет, не вводить
- ❌ Новый CSS-файл ради одной утилиты (это случай Tailwind)
- ❌ Tailwind для целой большой секции (это случай CSS-split)
- ❌ Дублирование одной задачи двумя способами

---

## Если удаляется страница / фича

Найти все CSS-файлы привязанные к фиче и **удалить вместе с кодом**. Иначе остаётся мёртвый код в bundle. Также удалить соответствующий `@import` из `src/app/globals.css`.
